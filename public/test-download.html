How it works:
When you click the download buttons above, your browser will:
  1. Make a request to the /config endpoint with the API key
  2. Receive the response with Content-Disposition: attachment header
  3. Automatically download the .sxcu file to your Downloads folder
  4. The file can then be imported into ShareX

Why curl/scripts show JSON instead of downloading:

Command-line tools like curl and Node.js scripts don't "download" files like browsers do. They simply display the response content. The download behavior is a browser feature that responds to the Content-Disposition: attachment header.

In browsers: Files are downloaded to the Downloads folder
In curl/scripts: Content is displayed as text

Testing with curl:
To save the config file using curl, use the -O flag:
curl -O "http://localhost:3854/config?key=YOUR_API_KEY"
